# Decisions - Tour Booking Flow

## Architectural Choices
[Agents will append decisions here]

### Booking Idempotency Helper (`src/utils/booking-idempotency.ts`)
- **Decision**: Accept `MedusaContainer` as parameter (not the service directly) to match subscriber usage patterns where `container` is available from `SubscriberArgs`.
- **Decision**: Return `InferTypeOf<typeof TourBooking>[]` to preserve full type safety on the returned booking records.
- **Decision**: Use `listTourBookings({ order_id })` which leverages the existing index on `order_id` (defined in the TourBooking model) for efficient lookups.
- **Decision**: No try/catch in the helper — let errors propagate to the caller (subscriber) which should handle logging/retry logic at its own level.
- **Decision**: Return the full array rather than a boolean, so callers can inspect existing bookings (e.g., for logging or status checks) beyond simple existence checks.

### Order Placed Subscriber (`src/subscribers/order-placed.ts`)
- **Decision**: Use `query.graph` with `entity: "order"` and `fields: ["id", "items.*"]` to fetch order items with metadata in a single query.
- **Decision**: Filter tour items by checking `item.metadata?.is_tour === true` (matches cart metadata structure from Task 2).
- **Decision**: Store `line_items` as a JSON object with `item_id`, `variant_id`, `title`, `quantity`, and `passengers` array — preserves all relevant order item data for the booking record.
- **Decision**: Use `createTourBookings()` (auto-generated by MedusaService) with an array of booking objects for batch creation.
- **Decision**: Wrap entire subscriber body in try/catch with `console.error` — prevents subscriber failures from crashing the event system while still logging errors.
- **Decision**: Pass `tour_id` from `item.metadata.tour_id` to the `tour_id` field (MedusaService's `belongsTo` convention maps `tour` relation to `tour_id` on create).
- **Decision**: Status defaults to `"pending"` explicitly in the create payload for clarity, even though the model default is also `"pending"`.
